# TinyML Vibration Fault Detection
> ESP32 + MPU6050 + Edge Impulse — Real-time vibration classification on the edge

---

## Overview

This project implements a complete TinyML pipeline that reads 3-axis accelerometer data from an MPU6050 sensor, extracts vibration features, and classifies machine state as **normal** or **fault** in real time on an ESP32 Dev Kit V1.

---

## Hardware Requirements

| Component | Detail |
|---|---|
| Microcontroller | ESP32 Dev Kit V1 |
| Sensor | MPU6050 (3-axis accelerometer) |
| Connection | I2C — SDA: GPIO 21, SCL: GPIO 22 |
| Power | 3.3V |

**Wiring:**
```
MPU6050   →   ESP32
VCC       →   3.3V
GND       →   GND
SDA       →   GPIO 21
SCL       →   GPIO 22
```

## Step-by-Step Usage

### Step 1 — Flash the Data Logger

1. Open `vibration_logger.ino` in Arduino IDE
2. Select board: **ESP32 Dev Module**
3. Flash to ESP32

### Step 2 — Record Normal State Data

```bash
# In vibration_serial_logger.py, set:
LABEL = "normal"
PORT  = "COM16"   # change to your port

python vibration_serial_logger.py
# Run for ~2 minutes → gives ~120 files
# Ctrl+C to stop
```

### Step 3 — Record Fault State Data

```bash
# Change machine state (add load, loosen bolt, etc.)
# In vibration_serial_logger.py, set:
LABEL = "fault"

python vibration_serial_logger.py
# Run for ~2 minutes
# Ctrl+C to stop
```

### Step 4 — Verify Your Data

```bash
python vibration_analysis.py vibration_data/normal_0000.csv
python vibration_analysis.py vibration_data/fault_0000.csv
```

Check that:
- Time domain signal looks clean (no flat lines)
- FFT shows a visible peak
- Kurtosis < 3 for normal, potentially > 3 for fault

### Step 5 — Prepare for Edge Impulse

```bash
# Fix CSV headers (required by Edge Impulse)
python fix_headers.py

# Reset timestamps to start from 0 (required by Edge Impulse)
python fix_timestamps.py
```

### Step 6 — Train on Edge Impulse

1. Create project at [edgeimpulse.com](https://edgeimpulse.com)
2. **Data Acquisition** → Upload existing data
   - Upload `normal_*.csv` files → label: `normal`
   - Upload `fault_*.csv` files → label: `fault`
3. **Create Impulse**
   - Window size: `1000 ms`
   - Window increase: `500 ms`
   - Processing block: **Spectral Analysis**
   - Learning block: **Classification**
4. **Spectral Features** → Save parameters → Generate features
5. **Classifier** → Start Training
6. **Model Testing** → Classify all → Verify accuracy > 90%
7. **Deployment** → Arduino Library → EON Compiler → Build → Download ZIP

### Step 7 — Deploy to ESP32

1. Arduino IDE → Sketch → Include Library → Add .ZIP Library → select downloaded ZIP
2. Open `vibration_inference.ino`
3. Flash to ESP32
4. Open Serial Monitor at **115200 baud**

## Troubleshooting

| Problem | Fix |
|---|---|
| `ERR: Sensor axes don't match model` | Make sure sensor names in sketch are `ax`, `ay`, `az` (not `accX`) |
| `Impulse window is empty` | Run `fix_timestamps.py` then re-upload to Edge Impulse |
| `Could not parse CSV` | Run `fix_headers.py` to rename `timestamp_ms` → `timestamp` |
| `UNCERTAIN` result | Remove the `CONVERT_G_TO_MS2` multiplication in `poll_IMU()` — keep data in g units |
| Low confidence | Make sure fault state has clearly different vibration from normal |
